---
title: "Error Events"
description: "Error notifications and status alerts"
---

Error events notify you of system errors and status issues that may affect the session. These events can occur at any time during the conversation lifecycle.

## Event Types

### `session.gpu_full`
Triggered when GPU resources are exhausted on the server.

**When to handle**: Inform the user and suggest retrying later.

<CodeGroup>
```javascript JavaScript
case "session.gpu_full":
  showError("The GPU resources are full. Please try again later!");
  break;
```

```python Python
if data.get('type') == 'session.gpu_full':
    show_error("The GPU resources are full. Please try again later!")
```
</CodeGroup>

### `session.insufficient_balance`
Triggered when account balance is insufficient to continue the service.

**When to handle**: Prompt user to recharge their account.

<CodeGroup>
```javascript JavaScript
case "session.insufficient_balance":
  showError("Insufficient balance, service has stopped, please recharge!");
  break;
```

```python Python
if data.get('type') == 'session.insufficient_balance':
    show_error("Insufficient balance, service has stopped, please recharge!")
```
</CodeGroup>

### `session.connection_limit_exceeded`
Triggered when the maximum number of concurrent connections is exceeded.

**Event data**:
- `message`: Error message describing the limit

<CodeGroup>
```javascript JavaScript
case "session.connection_limit_exceeded":
  showError(data.message);
  break;
```

```python Python
if data.get('type') == 'session.connection_limit_exceeded':
    show_error(data.get('message'))
```
</CodeGroup>

### `session.backend.error`
Triggered when a backend error occurs.

**Event data**:
- `message`: Error message from the backend

<CodeGroup>
```javascript JavaScript
case "session.backend.error":
  showError(data.message);
  break;
```

```python Python
if data.get('type') == 'session.backend.error':
    show_error(data.get('message'))
```
</CodeGroup>
